# Day 12 – Penetration Testing (Visual Diagram)

## Penetration Testing Flow

```
Permission Given
|
v
Simulated Attack
|
v
Test Exploitation
|
v
Measure Real Impact
|
v
Improve System Security

```
---

## Vulnerability Assessment vs Penetration Testing

```
Vulnerability Assessment
│
├── Finds weaknesses
├── No exploitation
└── Risk identification

Penetration Testing
│
├── Tests exploitation
├── Simulates attacks
└── Measures impact

```
---

## Ethical Rule in Penetration Testing

```
No Permission
|
v
Illegal Activity

Permission Given
|
v
Ethical & Legal Testing

```
